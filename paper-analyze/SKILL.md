# paper-analyze

## Role
A specialized Skill that analyzes arXiv papers from paper-search output, ranks them by relevance, and extracts key contributions and methods for each paper.

## Input/Output Specifications

### Input
Paper Metadata List from paper-search (Member A):
- Location: `/data/papers.json`
- Contains: `title`, `authors`, `abstract`, `arxiv_id`, `published`, `category`

### Output

**1. Console Output (Dialog Display)**
Ranked paper list displayed directly in the conversation:
```
## Paper Analysis Results

### Ranked by Relevance

1. **[Paper Title 1]** (Relevance: High)
   - Key Contributions: [contribution 1], [contribution 2]
   - Methods: [method 1], [method 2]

2. **[Paper Title 2]** (Relevance: Medium)
   ...
```

**2. JSON File Output**
- Location: `/data/paper_analysis.json`
```json
{
  "analysis_date": "YYYY-MM-DD",
  "total_papers": 10,
  "ranked_papers": [
    {
      "rank": 1,
      "title": "Paper Title",
      "arxiv_id": "2604.xxxxx",
      "relevance_score": 9,
      "key_contributions": ["contribution 1", "contribution 2"],
      "methods": ["method 1", "method 2"]
    }
  ]
}
```

## Workflow

### Step 1: Read Input from paper-search
Read the paper metadata from `/data/papers.json` generated by paper-search.

### Step 2: Relevance-Based Ranking
Analyze each paper's relevance to the search query based on:
- Title keyword match
- Abstract semantic similarity
- Category alignment
- Recency of publication

**Relevance Scoring (1-10):**
| Score | Description |
|-------|-------------|
| 9-10 | Highly relevant, strong keyword/semantic match |
| 7-8 | Relevant, partial match |
| 5-6 | Moderately relevant |
| 3-4 | Tangentially related |
| 1-2 | Low relevance |

### Step 3: Extract Key Contributions
For each paper, identify:
- **Key Contributions**: Main innovations, breakthroughs, or significant findings
- **Methods**: Techniques, architectures, algorithms used

### Step 4: Generate Console Display
Display the ranked list directly in the conversation with:
- Rank number
- Paper title
- Relevance score
- Key contributions (1-3 bullet points)
- Methods (1-3 bullet points)

### Step 5: Save to JSON File
Store the complete analysis in `/data/paper_analysis.json`:

```
/data/
├── papers.json           # From paper-search
└── paper_analysis.json   # Analysis results (THIS SKILL)
```

## Trigger Scenarios

**Automatic Flow:**
- Triggered after paper-search completes and user requests analysis

**Specific Queries:**
- "analyze these papers"
- "rank papers by relevance"
- "extract key contributions and methods"

## Error Handling

| Error | Handling |
|-------|----------|
| Input file not found | Request paper-search to run first |
| Empty paper list | Inform user and suggest new search |
| LLM analysis timeout | Retry once, then skip paper |
| Invalid metadata | Skip invalid entries, report count |

## Integration with Other Skills

- **Input Source**: `paper-search` (Member A) → `/data/papers.json`
- **Output Consumer**: `briefing-report-generator` (Member C)
- **Data Layer**: `/data/paper_analysis.json`
